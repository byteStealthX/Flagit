# Railway Deployment Crashed - Debugging Steps

## Status: CRASHED

### Check Deployment Logs (Do This Now!)

1. **In Railway Dashboard**:
   - Click on the crashed Flagit deployment
   - Look for **"Deploy Logs"** or **"Build Logs"** tab
   - Scroll to the bottom to see the error

2. **Common Errors**:

   **Missing npm dependencies**:
   ```
   Error: Cannot find module 'X'
   ```
   → Fix: Check package.json, run `npm install` locally

   **Environment variable missing**:
   ```
   Error: OPENAI_API_KEY is not defined
   ```
   → Fix: Double-check all 5 variables are added

   **Port binding error**:
   ```
   Error: listen EADDRINUSE
   ```
   → Fix: Railway auto-assigns PORT, check `process.env.PORT`

   **Syntax error in code**:
   ```
   SyntaxError: Unexpected token
   ```
   → Fix: Check server.js for errors

### Quick Fix Steps

**If environment variables weren't saved**:
1. Go to Variables tab
2. Re-add all 5 variables from `DEPLOY_NOW.txt`
3. Click "Deploy" → "Redeploy"

**If there's a code error**:
1. Check the error message
2. Fix in local code
3. Commit: `git add . && git commit -m "fix: railway deployment error"`
4. Push: `git push origin main`
5. Railway auto-redeploys

**Force Redeploy**:
1. Railway → Deployments tab
2. Click "..." on latest deployment
3. Click "Redeploy"
